# Отчет по учебному проекту
## Сервис рабочих программ (Вариант 13)

### Функционал сервиса
* Регистрация и хранение метаданных рабочих программ дисциплин (РПД).
* Привязка программ к внешним специальностям и дисциплинам через транзитивную таблицу назначений.
* Сервис не хранит справочники специальностей, используя только их идентификаторы (интеграция с Specialty Service).
* Учет версий программ и путей к файлам.

---

### 1. Добавить рабочую программу
| Параметр | Пояснение | Обязательность | Тип | Ограничение | Значение по умолчанию |
| :--- | :--- | :--- | :--- | :--- | :--- |
| title | Название программы | Да | string | Not NULL | — |
| file_path | Путь к файлу на сервере | Да | string | Not NULL | — |
| version | Версия документа | Да | string | Формат x.x | "1.0" |

**Возвращаемое значение:** Объект программы с присвоенным `id` и `created_at`.

---

### 2. Создать назначение программы (Assignment)
Используется для привязки программы к специальности и дисциплине.
| Параметр | Пояснение | Обязательность | Тип |
| :--- | :--- | :--- | :--- | 
| program_id | ID программы | Да | int |
| specialty_id | ID из внешнего сервиса специальностей | Да | int |
| discipline_id | ID из внешнего сервиса дисциплин | Да | int |

---

### 3. Изменить программу по ID
| Параметр | Пояснение | Обязательность | Тип | Ограничение |
| :--- | :--- | :--- | :--- | :--- |
| title | Новое название | Нет | string | Not NULL |
| file_path | Новое расположение | Нет | string | Not NULL |

---

### 4. Удаление программы по ID
Вернет **True**, если запись и связанные назначения удалены, иначе **False**.

---

### 5. Получить список программ по фильтрам
| Параметр | Пояснение | Тип |
| :--- | :--- | :--- |
| specialty_id | Фильтр по ID специальности | int |
| discipline_id | Фильтр по ID дисциплины | int |
| version | Точное совпадение версии | string |

---

### ER-диаграмма
Диаграмма соответствует Третьей Нормальной Форме (3НФ). 
1. Сущность **WorkProgram** содержит только данные о самой программе.
2. Сущность **ProgramAssignment** является транзитивной таблицей, реализующей связь "многие-ко-многим".
3. Данные о специальностях и дисциплинах являются внешними (Master Data) и подключаются по ID.
